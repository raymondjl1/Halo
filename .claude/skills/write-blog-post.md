# Write Blog Post

## Before Starting
Read `documents/halo_v2_business_plan.md` Section 11 (The Pitch) and `docs/style-guide.md` for voice/tone.

## Steps

1. **Define topic and angle**: What's the takeaway for the reader? What search query would find this?

2. **Write in markdown** at `documents/marketing/blog/{slug}.md`

3. **Follow voice rules**:
   - Conversational, first-person ("I built this because...")
   - Honest about tradeoffs — don't oversell
   - Lead with the business outcome, not the technology
   - Say "workflow" not "process", "reliable" not "deterministic"
   - Never name competitors by brand — compare features

4. **Structure**:
   - Headline (benefit-focused, under 70 chars)
   - Hook paragraph (the problem)
   - Body (the solution, with concrete examples)
   - CTA (try Halo free, link to signup)
   - Keep under 1,000 words

5. **SEO**: Include target keyword in title, first paragraph, and one subheading.

6. **Submit for review** — Jeff approves all customer-facing content before publish.

7. **Publish** via Odoo blog module once approved.
